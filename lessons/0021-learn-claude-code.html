一句话说清楚

Learn Claude Code 是一套开源课程，教你从零开始、一步步搭建一个「迷你版 Claude Code」。它把 AI 编程助手拆成 20 个小步骤，每步只加一个机制——从最基础的「叫模型干活」开始，一路叠加到「多个 AI 组队协作」。它不是给你用的工具，而是帮你理解工具是怎么造出来的。

这到底是个什么东西？

它不是 App，不是工具——它是一套开源教材。仓库里 20 个文件夹（s01→s20），每个里面一篇教程 + 能跑的示例代码。配套网站 learn.shareai.run 把内容做成了交互式网页。

教学目标是让你理解 Claude Code 这种 AI 编程助手内部怎么运作的。不是读论文，是自己动手写一个迷你版出来。

5 层架构，20 步渐进

🔧 第 1 层：工具与执行（s01–s04）

Agent 循环 → 工具注册 → 权限控制 → 钩子系统

目标：先让 AI 能「做事情」——调用工具、读写文件

🧠 第 2 层：规划与控制（s05–s08）

计划工具 → 子代理 → 技能加载 → 上下文压缩

目标：让 AI 能「处理复杂任务」——拆解目标、协调子任务

💾 第 3 层：记忆与恢复（s09–s11）

跨会话记忆 → 系统提示组装 → 错误恢复

目标：让 AI 能「记住东西 + 出错自救」

⚡ 第 4 层：并发与调度（s12–s14）

任务系统 → 后台任务 → 定时任务

目标：让 AI 能「同时干多件事」

🤝 第 5 层：多 Agent 平台（s15–s20）

Agent 团队 → 团队协议 → 自主 Agent → 工作树隔离 → MCP 插件 → 综合 Agent

目标：让多个 AI「组队协作」，一个当主管、其他当专家

一个「教程」凭什么 69.5K 星？

🎯 踩中了 2026 最大的好奇心缺口

Claude Code、Cursor、Copilot……所有人都在用 AI 写代码，但几乎没人知道它们内部怎么运转。这个课程恰好回答了「你天天用的东西到底怎么做的」。

🧱 「每次只加一块积木」的教学设计

不做大而全的架构图，而是让你一行行加代码，每加一行都能跑、都能看到效果。从 102 行代码的最小 Agent 开始，20 步后手里是一个完整的 AI 编程助手。

🌐 中英日三语 + 交互式网站

不只是 GitHub 上的 Markdown，还配了完整的教学网站——代码高亮、步骤动画、多语言切换。投入度远超普通开源项目。

📣 自带传播属性

学完的人会自发发推——「我用这个课程写了一个迷你 Claude Code」本身就是绝佳的内容营销。

💡 做独立开发的启发

📖 「教程」可以是比工具更成功的开源产品

Learn Claude Code 的星数超过了绝大多数工具类项目。内容型产品天然有故事性——容易写推文、做视频、引发讨论。如果你在某个领域有深度 know-how，把它做成开源教程可能比做成工具传播力更强。

🪜 「渐进式复杂度」是产品设计的万能法则

「从最小可用版本开始，每次只加一个机制，每一步都能跑」——这套哲学可以平移到你的 App onboarding 里。用户第一次打开产品，别一股脑展示所有功能，先完成一个最小闭环，再逐步解锁新能力。

🔍 「消除黑箱焦虑」是隐性刚需

为什么 69.5K 人想了解 Claude Code 内部？因为大家天天用 AI，但心里没底——它怎么决策的？会不会漏掉信息？你做任何产品，如果用户「依赖它但不理解它」，就有做透明化解说的价值。

💡 对返利的启发

你的代理每天都在用转链工具，但他们真的理解「转链」是怎么回事吗？不理解就没法排查问题、没法跟用户解释。做一个「转链是怎么工作的」5 分钟图解，放在 App 里，可能就是减少客服压力的杠杆点。
